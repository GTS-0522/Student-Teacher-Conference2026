# 二者面談 日程調整サイト（2026年8月）

スマートフォン向けの、教師用・生徒用を分離した二者面談日程調整サイトです。

## フォルダ構成

```text
futari-mendan-github/
├─ teacher-site/   教師用GitHub Pages（予定の追加・編集・削除）
├─ student-site/   生徒用GitHub Pages（閲覧・希望入力）
└─ gas-backend/    共通データ・メール送信・スプレッドシート記録
```

教師用と生徒用は、別々のGitHubリポジトリとして公開します。画面間を移動するリンクはありません。

## 最初の設定（約15分）

### 1. GASを準備する

1. [Google Apps Script](https://script.google.com/)を開き、「新しいプロジェクト」を作成します。
2. `gas-backend/Code.gs` の内容を、GASの `コード.gs` にすべて貼り付けます。
3. プロジェクトの設定を開き、タイムゾーンを「(GMT+09:00) 東京」にします。
4. エディタ上部で関数 `setup` を選び、「実行」を押します。初回の権限を許可してください。
5. エディタ上部で関数 `setTeacherPassword` を選び、「実行」を押します。
6. 左側の「プロジェクトの設定」→「スクリプト プロパティ」で、`TEACHER_PASSWORD` の値を自分だけが知る管理パスワードへ変更します。
7. 「デプロイ」→「新しいデプロイ」→種類「ウェブアプリ」を選びます。
8. 「次のユーザーとして実行」は自分、「アクセスできるユーザー」は全員にし、デプロイします。
9. 表示された `/exec` で終わるウェブアプリURLをコピーします。

> 「全員」にするのは、生徒の端末から予定を読んだり希望を送ったりするためです。予定の追加・変更・削除には、別途管理パスワードが必要です。

### 2. 2つのサイトにGAS URLを設定する

今回の修正版には、現在使用中のGAS URLを設定済みです。新しいGASプロジェクトや別のデプロイURLへ変更した場合だけ、次の2ファイルのURLを置き換えます。

- `teacher-site/config.js`
- `student-site/config.js`

### 3. 教師用GitHub Pagesを公開する

1. GitHubで教師用の新しいリポジトリを作ります（例：`mendan-teacher-2026`）。
2. `teacher-site` フォルダ内のファイルだけを、リポジトリ直下へアップロードします。
3. リポジトリの「Settings」→「Pages」を開きます。
4. Sourceを「Deploy from a branch」、Branchを `main` / `(root)` にして保存します。

### 4. 生徒用GitHub Pagesを公開する

1. GitHubで生徒用の別リポジトリを作ります（例：`mendan-student-2026`）。
2. `student-site` フォルダ内のファイルだけを、リポジトリ直下へアップロードします。
3. 教師用と同様にGitHub Pagesを有効にします。

## 使い方

### 教師用

- 空いている時間帯、または「予定を追加」を押すと予定を登録できます。
- 「二者面談」と「教師予定」を選べます。
- 予定カードを押すと編集・削除できます。
- 最初の登録・編集時に管理パスワードを入力します。パスワードはそのタブを閉じるまでだけ保持されます。
- 20秒ごとに最新データを自動取得します。

### 生徒用

- 教師用で登録した予定を閲覧できます。
- 30秒ごとに最新データを自動取得します。
- 「希望入力」から氏名と希望日時を送信できます。
- 送信内容は指定スプレッドシートに記録され、`yusuke.tigers.0522@gmail.com` にメールされます。
- 希望入力は2026年8月10日（月）まで受け付け、8月11日以降は入力画面を自動で閉じます。
- スマートフォンでは、読み込み失敗時に最大3回まで自動再試行します。

## 補足

- 対象期間は2026年8月17日（月）〜8月28日（金）の平日、9:00〜16:30です。
- 開始・終了時刻は5分単位です。
- 2026年8月17日より前は8月17日、8月17日〜21日は当日、8月24日以降は8月24日が左端になります。
- 過ぎた日は薄いグレーになりますが、文字と予定は確認できます。
- スプレッドシートには「予定」「希望」の2シートが自動で作られます。
- GASコードを変更した場合は、「デプロイを管理」から新しいバージョンとして再デプロイしてください。
- スマートフォンで通信に失敗した場合は、同じタブで前回読み込めた予定を一時表示します。タブを閉じると一時データは消えます。
